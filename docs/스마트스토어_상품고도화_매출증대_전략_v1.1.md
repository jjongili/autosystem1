## 1. 핵심 개념: 노출-유입-전환 파이프라인

### 1.1 퍼널 구조 이해

```
┌─────────────────────────────────────────────────────────────────────┐
│                        매출 = 노출 × 클릭률 × 전환율 × 객단가            │
└─────────────────────────────────────────────────────────────────────┘

[노출] ───→ [클릭] ───→ [유입] ───→ [체류] ───→ [구매] ───→ [재구매]
  │            │          │          │          │           │
  │            │          │          │          │           │
  ▼            ▼          ▼          ▼          ▼           ▼
적합도       썸네일     상세페이지   콘텐츠     가격/혜택   CRM
인기도      상품명      첫인상      품질       리뷰/신뢰   단골관리
신뢰도      가격표시    로딩속도    정보력     쿠폰/찜     알림발송
```

### 1.2 지표 정의

| 지표 | 정의 | 확인 위치 |
|------|------|-----------|
| **노출수** | 검색결과에서 상품이 보여진 횟수 | 쇼핑파트너센터 → 상품리포트 → 상품클릭리포트 |
| **클릭수** | 노출된 상품을 클릭한 횟수 | 쇼핑파트너센터 → 상품리포트 → 상품클릭리포트 |
| **유입수** | 스토어로 실제 방문한 고객 수 | 스마트스토어센터 → 데이터분석 → 마케팅분석 |
| **전환수** | 실제 구매로 이어진 건수 | 스마트스토어센터 → 데이터분석 → 판매분석 |
| **전환율** | 유입 대비 구매 비율 (업계 평균 2~3%) | 통계 → 쇼핑행동분석 |

### 1.3 성장 플라이휠 (선순환 구조)

```
키워드 최적화 → 노출 상승 → 클릭수 증가 → 유입 증가
       ↑                                        ↓
   매출 증가 ← 인기도 상승 ← 전환율 향상 ← 구매 증가
```

**핵심**: 유입이 증가하면 매출이 늘고, 매출이 늘면 인기도가 상승하고, 인기도가 상승하면 노출이 더 증가하는 선순환

## 2. 네이버 쇼핑 검색 알고리즘 심층 분석

### 2.1 랭킹 점수 구성

```
┌─────────────────────────────────────────────────────────────────────┐
│         노출 순위 = 랭킹점수 = 적합도 + 인기도 + 신뢰도              │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 적합도 (Relevance Score)

**정의**: 소비자의 검색 의도와 상품 정보가 얼마나 일치하는가

**구성요소 (가중치 순서)**:
1. **상품명** ← 가장 중요 (검색어 매칭의 핵심)
2. **스토어명**
3. **브랜드/제조사**
4. **카테고리**
5. **속성**
6. **태그**

**적합도 최적화 전략**:

| 요소 | 최적화 방법 |
|------|-------------|
| 상품명 | 메인키워드 + 세부키워드 조합, 50자 이내 권장, 동일 단어 3회 이상 반복 금지 |
| 카테고리 | 실제 상품과 정확히 일치하는 리프 카테고리 선택 |
| 브랜드 | 정식 브랜드명 등록 (상품명보다 브랜드 필드 우선 인식) |
| 속성 | 모든 필수 속성 + 선택 속성 최대한 입력 |
| 태그 | 연관 검색어 기반 태그 5~10개 설정 |

**상품명 작성 공식**:
```
[브랜드] + [핵심키워드] + [세부키워드] + [스펙/특징]

예시: 삼성 갤럭시버즈3 프로 블루투스 이어폰 노이즈캔슬링 화이트
```

**금지 사항**:
- 동일 단어 3회 이상 반복
- 경쟁사 브랜드명 도용
- 과도한 특수문자 사용
- 허위/과장 키워드

### 2.3 인기도 (Popularity Score)

**정의**: 해당 상품이 소비자들에게 얼마나 인기 있는가

**구성요소**:

| 요소 | 측정 기간 | 설명 |
|------|-----------|------|
| **클릭수** | 최근 7일 | 쇼핑검색에서 발생한 클릭 |
| **판매실적** | 최근 2일/7일/30일 | 네이버페이 통한 판매수량/금액 지수화 |
| **리뷰수** | 누적 | 카테고리별 가중치 상이 |
| **찜수** | 누적 | 상품찜 + 스토어찜 |
| **최신성** | 등록일 기준 | 신상품 한시적 노출 부스트 (3~7일) |

**⚠️ 중요**: 네이버 가격비교(쇼핑검색)를 거치지 않은 구매(직접유입, 외부링크)는 인기도 점수에 영향 없음

**인기도 빠르게 올리는 방법**:
1. 옵션 수 증가 + 키워드 옵션 구성
2. 썸네일/상세페이지 최적화 → CTR 상승
3. 긍정적 리뷰 확보 (포토리뷰 우선)
4. 찜 + 쿠폰 조합 이벤트
5. 쇼핑검색광고 활용 → 초기 클릭/구매 확보

**최신성 버프 활용**:
- 신상품 등록 후 3~7일간 노출 점수 가점
- 이 기간 동안 판매 집중 → 인기도 점수 극대화
- 광고 집행과 병행 시 효과 배가

### 2.4 신뢰도 (Trust Score)

**정의**: 해당 상품/판매자가 소비자에게 신뢰를 줄 수 있는가

**구성요소**:

| 요소 | 설명 |
|------|------|
| **상품명 SEO 스코어** | 네이버 가이드라인 위반 시 강력한 페널티 |
| **네이버 쇼핑 페널티** | 어뷰징 적발 시 부과 |
| **판매자 등급** | 새싹/씨앗/파워/빅파워/프리미엄 |
| **고객 혜택** | 쿠폰, 적립금, 무료배송 설정 여부 |

**페널티 점수 기준 (2024.07.24~ 적용)**:

| 항목 | 점수 |
|------|------|
| 품절 취소 | 2점 |
| 선물하기 품절 취소 | 3점 |
| 반품처리 지연 (3영업일+) | 1점 |
| 교환처리 지연 (3영업일+) | 1점 |
| 재교환처리 지연 | 1점 |

**⚠️ 제재 기준**: 최근 30일간 페널티 10점 이상 + 페널티 비율 40% 이상 시 판매 활동 제한

## 3. 중국 구매대행 특화 전략

### 3.1 구매대행 vs 일반 판매 차이점

| 구분 | 일반 판매 (사입) | 중국 구매대행 |
|------|------------------|---------------|
| **등록 방식** | 수동 소량등록 | 반자동 대량등록 |
| **상품 수** | 수십~수백 개 | 수천~수십만 개 |
| **브랜드** | 브랜드 보유 | 브랜드 없음 (노브랜드) |
| **키워드 전략** | 대형 키워드 공략 | 롱테일/틈새 키워드 공략 |
| **경쟁 구도** | 국내 사입 상품과 경쟁 | 틈새시장 선점 |
| **마진 구조** | 높은 마진 | 낮은 마진 × 대량 판매 |
| **재고 리스크** | 있음 | 없음 (주문 후 발주) |

### 3.2 구매대행 상품명 로직 (핵심)

**일반 상품명 공식** (사입):
```
[브랜드] + [핵심키워드] + [세부키워드] + [스펙/특징]
예: 삼성 갤럭시버즈3 프로 블루투스 이어폰 노이즈캔슬링 화이트
```

**⚠️ 구매대행 상품명 공식** (브랜드 없음):
```
[메인키워드] + [세부키워드] + [스펙] + [특징]
예: 캠핑 접이식 의자 경량 알루미늄 야외 낚시 휴대용 백패킹 차박
```

### 3.3 구매대행 키워드 전략

#### 왜 롱테일 키워드인가?

```
┌─────────────────────────────────────────────────────────────────────┐
│  대형 키워드: "캠핑의자" (100만 검색)                                │
│  → 국내 사입 브랜드 상품이 상위 독점                                │
│  → 광고비 과다, 진입 불가능                                        │
│                                                                     │
│  롱테일 키워드: "초경량 백패킹 접이식 의자" (1천 검색)               │
│  → 경쟁 상품 적음                                                  │
│  → 실구매자가 검색하는 키워드                                      │
│  → 전환율 높음 (구매 의도 명확)                                    │
└─────────────────────────────────────────────────────────────────────┘
```

#### 키워드 피라미드 전략

```
                    ┌───────────┐
                    │  초대형   │  ← 진입 금지 (국내 사입 영역)
                    │ 캠핑의자  │     검색량 10만+
                    └─────┬─────┘
                          │
                ┌─────────┴─────────┐
                │      대형         │  ← 신중하게 접근
                │ 접이식 캠핑의자   │     검색량 1~10만
                └─────────┬─────────┘
                          │
        ┌─────────────────┴─────────────────┐
        │              중형                  │  ← 주력 공략 대상
        │   경량 접이식 캠핑의자 휴대용       │     검색량 1천~1만
        └─────────────────┬─────────────────┘
                          │
    ┌─────────────────────┴─────────────────────┐
    │                  소형/롱테일                │  ← 대량 등록 핵심
    │  초경량 백패킹 접이식 의자 알루미늄 차박    │     검색량 100~1천
    └───────────────────────────────────────────┘
```

#### 실구매자 키워드 특징

| 키워드 유형 | 예시 | 전환율 |
|-------------|------|--------|
| 일반 검색 | "캠핑의자" | 1% 이하 |
| 구체적 검색 | "접이식 캠핑의자" | 2~3% |
| **실구매자 검색** | "초경량 백패킹 의자 1kg 이하" | **5%+** |
| **문제해결 검색** | "허리 안아픈 캠핑의자" | **7%+** |

### 3.4 구매대행 상품명 작성 실전 가이드

#### 기본 구조
```
[메인키워드] + [세부키워드1] + [세부키워드2] + [스펙] + [특징/용도]

※ 브랜드 필드: 비워두거나 "노브랜드" 입력
※ 제조사 필드: 비워두거나 "중국" 입력
```

#### Good vs Bad 예시

**❌ Bad (일반적 접근)**:
```
여성 원피스 봄 원피스 여름 원피스 가을 원피스 원피스
→ 키워드 나열, 반복, 대형 키워드만 사용
```

**✅ Good (구매대행 최적화)**:
```
플리츠 롱원피스 하객룩 여성 봄신상 오피스룩 하늘색 A라인
→ 세부키워드 조합, 실구매자 검색어 반영, 스펙(A라인) 포함
```

**❌ Bad (대형 키워드)**:
```
블루투스 이어폰 무선이어폰 가성비 이어폰
→ 국내 브랜드와 경쟁 불가
```

**✅ Good (롱테일 공략)**:
```
오픈형 무선이어폰 귀안아픈 운동용 방수 골전도 러닝 IPX7
→ 실구매자 니즈 반영 (귀안아픈), 스펙(IPX7), 용도(러닝)
```

### 3.5 대량등록 시 상품명 자동 생성 로직

#### 상품명 템플릿 구조
```python
# 상품명 자동 생성 공식
product_title = f"{main_keyword} {sub_keyword1} {sub_keyword2} {spec} {feature} {color}"

# 예시 템플릿
templates = {
    "의류": "{스타일} {아이템} {성별} {시즌} {핏} {소재} {컬러}",
    "가전": "{기능} {아이템} {용도} {스펙} {특징} {컬러}",
    "생활용품": "{용도} {아이템} {재질} {사이즈} {특징} {컬러}",
    "패션잡화": "{스타일} {아이템} {성별} {용도} {소재} {컬러}",
}
```

#### 키워드 조합 우선순위

| 순서 | 요소 | 예시 |
|------|------|------|
| 1 | 메인키워드 | 접이식의자, 무선이어폰 |
| 2 | 세부키워드 (용도) | 캠핑, 낚시, 차박, 운동용 |
| 3 | 세부키워드 (특성) | 경량, 휴대용, 방수 |
| 4 | 스펙 | 알루미늄, 500g, IPX7 |
| 5 | 특징/차별점 | 허리지지, 귀안아픈 |
| 6 | 컬러/사이즈 | 블랙, L사이즈 |

### 3.6 의심 키워드 필터링 (클린위반 방지)

**자동 필터링 필요 키워드**:

| 유형 | 예시 | 위험도 |
|------|------|--------|

## 4. 데이터 확인 위치 및 KPI 관리

### 4.1 핵심 데이터 확인 위치

#### 키워드별 결제금액
**위치**: 스마트스토어센터 → 데이터분석 → 판매분석 → 상품/검색채널
- 상품카테고리별 검색채널별 결제금액
- 결제금액 Top 10 상품의 키워드별 결제금액
- **활용**: 어떤 키워드가 실제 매출로 이어지는지 파악

#### 키워드별 유입수
**위치**: 스마트스토어센터 → 데이터분석 → 마케팅분석 → 상품노출성과
- 유입 Top10 상품별 평균노출순위 및 유입수
- **주의**: 네이버가격비교-통합검색채널은 노출순위 0으로 표시
- 유입수 기준 상위 500 품번만 제공

#### 클릭수 확인
**위치**: 쇼핑파트너센터 → 상품리포트 → 상품클릭리포트-모바일/PC → 상품별
- 상품ID, 상품명별 노출수, 클릭수, 클릭률, CPC수수료
- **핵심 지표**: 클릭률(CTR) = 클릭수 / 노출수 × 100

#### 마켓 유입수
**위치**: 스마트스토어센터 → 데이터분석 → 마케팅분석 → 전체채널
- 마케팅채널별 유입수(일별) 그래프
- 채널: 네이버 쇼핑검색광고, 네이버가격비교-검색, 네이버플러스스토어 등

#### 전환율 확인
**위치**: 스마트스토어센터 → 통계 → 쇼핑행동분석
- 상품별/페이지별 상세페이지 조회수
- 조회대비결제비율 (= 상품별 전환율)

### 4.2 KPI 관리 체계

#### 일간 점검
- [ ] 주문수 확인
- [ ] 유입수 추세 확인
- [ ] CS 미처리 건 확인
- [ ] 클릭률 변동 모니터링

#### 주간 점검
- [ ] 상품별 유입수 증감 분석 (20~30% 감소 시 물갈이 검토)
- [ ] 키워드별 결제금액 Top 10 변동 추적
- [ ] 클릭률 1% 이하 상품 개선 대상 선정
- [ ] 전환율 하락 추세 상품 분석

#### 월간 점검
- [ ] 채널별 기여도 분석
- [ ] 상품별 마진율 점검 (15% 이하 가격 재조정)
- [ ] 물갈이 대상 선정 (2개월 경과 + 하락 추세)
- [ ] 광고 ROAS 분석

## 5. 상품 고도화 5단계 전략 (대량등록 버전)

### 5.1 반자동 가공 5단계 공식 (구매대행 최적화)

```
┌─────────────────────────────────────────────────────────────────────┐
│   1단계: 데이터 수집/분석 → 2단계: 상위 상품 선별                     │
│   → 3단계: 상품명/키워드 재최적화 → 4단계: 이미지/상세 개선            │
│   → 5단계: Refresh 재업로드 (물갈이)                                 │
└─────────────────────────────────────────────────────────────────────┘
```

#### 1단계: 데이터 수집/분석 (대량 상품 기준)
- 전체 상품 유입수/조회수/클릭률 일괄 추출
- 판매 발생 상품 vs 미발생 상품 분류
- **상위 20% 상품** 리스트업 (파레토 법칙)

```python
# 대량 상품 분석 기준
상위_그룹 = {
    "기준": "유입수 상위 20% AND 판매 발생",
    "액션": "집중 가공 대상",
    "목표": "매출 80% 담당"
}

중간_그룹 = {
    "기준": "유입수 있으나 판매 미발생",
    "액션": "전환율 개선 대상",
    "목표": "전환율 2% 달성"
}

하위_그룹 = {
    "기준": "유입수 없음",
    "액션": "물갈이 또는 방치",
    "목표": "데이터 수집용"
}
```

#### 2단계: 상위 상품 선별
- **선별 기준**:
  - 유입수 상위 20%
  - 클릭률 1% 이상
  - 전환율 1% 이상 (또는 판매 발생)
  - 상승 추세 상품

- **제외 기준**:
  - 유입수 하락 추세 (주간 20~30% 감소)
  - 경쟁 심화 키워드 상품
  - 마진율 10% 이하 상품

#### 3단계: 상품명/키워드 재최적화 (구매대행 특화)
```
Before: 여성 원피스 봄 신상 여름 원피스 가을 원피스
After:  플리츠 롱원피스 하객룩 여성 봄신상 오피스룩 A라인

변경 포인트:
1. 대형 키워드 "원피스" 반복 제거
2. 실구매자 키워드 "하객룩", "오피스룩" 추가
3. 스펙 "A라인", "플리츠", "롱" 추가
4. 시즌 키워드 "봄신상" 위치 조정
```

**상품명 재작성 자동화 로직**:
```python
def optimize_title(original_title, category):
    # 1. 의심 단어 필터링
    filtered = remove_suspicious_words(original_title)
    
    # 2. 키워드 분석
    keywords = extract_keywords(filtered)
    main_kw = keywords['main']
    sub_kws = keywords['sub']
    specs = keywords['spec']
    
    # 3. 실구매자 키워드 추가
    buyer_kws = get_buyer_keywords(category, main_kw)
    
    # 4. 최적화 상품명 생성
    optimized = f"{main_kw} {' '.join(sub_kws[:2])} {' '.join(buyer_kws[:2])} {' '.join(specs[:2])}"
    
    return optimized[:50]  # 50자 제한
```

#### 4단계: 이미지/상세 개선
- **대표이미지**: 1~3장 교체 → 최신성 점수 갱신
- **상세페이지**: 
  - 중국 원본 이미지 그대로 사용 (대량등록 특성)
  - 상위 상품만 선별적으로 한글화/가공
- **구매유도 요소 추가**:
  - 쿠폰 배너
  - 리뷰 하이라이트
  - 배송 안내

#### 5단계: Refresh 재업로드 (물갈이)
- 업로드 날짜 최신화 → 최신성 버프 재활성화
- 신규 상품 ID 발급 (필요시)
- **⚠️ 주의**: 동일 상품 삭제 후 재등록은 어뷰징 간주

### 5.2 상품명 최적화 상세 가이드 (구매대행 특화)

**⚠️ 구매대행 상품명 vs 일반 상품명 차이**:

## 6. 물갈이(상품교체) 전략 (대량등록 버전)

### 6.1 상품 수명 주기

```
[상승기] ──→ [안정기] ──→ [하락기]
 1~4주        1~2개월      2개월 이후
   │             │             │
   ▼             ▼             ▼
신규업로드    노출/판매     유입/조회
효과로       일정 수준     전환율
노출 증가    유지          하락
```

### 6.2 물갈이 타이밍 판단 기준

**기본 주기**: 평균 2개월
**핵심**: 정답은 2개월이 아니라 **'유입수 하락 시점'**

#### 물갈이 필요 신호 5가지

| 신호 | 구체적 지표 |
|------|-------------|
| **유입수 감소** | 주간 유입량 20~30% 감소 |
| **조회수 하락** | 업로드 프로그램 조회수 지속 하락 |
| **키워드 경쟁 증가** | 동일 키워드 경쟁 상품 급증 |
| **리뷰 증가 속도 감소** | 주간 신규 리뷰 50% 이상 감소 |
| **전환율 하락** | 전환율 업계 평균(2%) 이하 |

#### 클릭률 기준 판단

| 클릭률 | 상태 | 조치 |
|--------|------|------|
| 2% 이상 | 우수 | 유지 |
| 1~2% | 보통 | 개선 검토 |
| 1% 이하 | 위험 | 즉시 물갈이 |

### 6.3 물갈이 실행 프로세스 (대량등록 버전)

```
1. 대상 선정
   └─→ 유입수 하락 + 전환율 하락 상품 리스트업

2. 데이터 백업
   └─→ 기존 상품 정보, 리뷰, 키워드 성과 기록

3. 상품 수정 (기존 상품 유지 시)
   └─→ 상품명 50% 이상 변경
   └─→ 대표이미지 전체 교체
```

### 8.4 인증 토큰 발급

**OAuth 2.0 기반 인증**:

```python
import requests
import time
import hmac
import hashlib
import base64

# 타임스탬프 생성
timestamp = str(int(time.time() * 1000))

# 시그니처 생성
message = f"{client_id}_{timestamp}"
signature = base64.b64encode(
    hmac.new(
        client_secret.encode('utf-8'),
        message.encode('utf-8'),
        hashlib.sha256
    ).digest()
).decode('utf-8')

# 토큰 요청
response = requests.post(
    'https://api.commerce.naver.com/external/v1/oauth2/token',
    data={
        'client_id': client_id,
        'timestamp': timestamp,
        'client_secret_sign': signature,
        'grant_type': 'client_credentials',
        'type': 'SELF'  # 판매자 본인 스토어
    }
)

access_token = response.json()['access_token']
```
